1) CRITICAL FUNCTIONAL ISSUES (Pages/Routes not working)

- Issue: Links to /resources/events/saviwealth-investment-summit-2025, but route is not defined in User_Client(User Frontend)/src/App.tsx.
- Impact: 404/navigation failure from Featured Event CTA.
- Improvement: Add route + detail page OR remove/deactivate link.

1.3 Media detail routes missing
- File: User_Client(User Frontend)/src/pages/ResourcesAudioVideoClips.tsx
- Issue: Links to /resources/media/complete-guide-mutual-funds and /resources/media/market-insights-weekly, but no matching routes in App.tsx.

- Improvement: Add route + detail pages OR change links to valid destinations.

1.4 Existing page file is unreachable by routing
- File: User_Client(User Frontend)/src/pages/Offerings.tsx
- Issue: Page exists but no route in App.tsx.
- Impact: Dead page file (not directly accessible).
- Improvement: Add /offerings route or remove/merge file.

====================================================
2) API / BACKEND INTEGRATION ISSUES
====================================================

2.1 CORS origin mismatch with documented run ports
- File: Backend/server.js
- Issue: CORS allows only http://localhost:5173 and http://localhost:3000.
- Reference docs mention User frontend running on port 5174.
- Impact: API calls from 5174 can fail due to CORS.
- Improvement: Include all actual frontend origins via env-driven CORS config.

2.2 Admin UI actions are mostly not connected to backend write APIs
- Files:
  - Admin_Client(Admin Frontend)/src/pages/Investments.tsx
  - Admin_Client(Admin Frontend)/src/pages/Advisors.tsx
  - Admin_Client(Admin Frontend)/src/pages/Notifications.tsx
  - Admin_Client(Admin Frontend)/src/pages/Settings.tsx
- Issue: Create/Edit/Delete/Save modals/buttons are rendered but mostly only update local UI state or no-op.
- Impact: UI appears functional but changes are not persisted.
- Improvement: Wire all action handlers to existing backend endpoints with optimistic updates + error handling.

2.3 Reports page expects analytics data shape not returned by backend
- File: Admin_Client(Admin Frontend)/src/pages/Reports.tsx
- Issue: Uses analytics.userGrowth and analytics.revenue for charts.
- Backend /api/admin/analytics currently returns totals/averages only (no time-series arrays).
- Impact: Empty charts or misleading output.
- Improvement: Align frontend chart model with backend response OR extend backend response.

2.4 Dashboard type field mismatch for investment distribution
- File: Admin_Client(Admin Frontend)/src/pages/Dashboard.tsx
- Issue: Chart groups investments by inv.type, but backend investment records use investmentType.
- Impact: Portfolio type chart can collapse into “Other”/incorrect categories.
- Improvement: Normalize field mapping to investmentType.

====================================================
3) SECURITY / PRODUCTION-READINESS ISSUES
====================================================

3.1 Sensitive defaults in backend source
- File: Backend/server.js
- Issue: Default DB password and default JWT secret are hardcoded fallbacks.
- Impact: Security risk if env vars are missing/misconfigured.
- Improvement: Fail fast when required secrets/env vars are missing; never hardcode credentials.

3.2 Test-only admin endpoint exposed in runtime app
- File: Backend/server.js
- Endpoint: POST /api/auth/create-admin
- Issue: Marked “for testing - remove in production”, but currently active.
- Impact: Unauthorized admin creation risk if misused.
- Improvement: Disable outside development or protect with strict admin bootstrap controls.

3.3 Weak default advisor password on creation
- File: Backend/server.js
- Endpoint: POST /api/admin/advisors
- Issue: Uses static password advisor123.
- Impact: Account compromise risk.
- Improvement: Generate secure random password + mandatory reset flow.

3.4 Excessive internal error disclosure
- File: Backend/server.js (login error handling)
- Issue: Sends raw DB/internal error details in API response in some paths.
- Impact: Information leakage.
- Improvement: Return generic client-safe messages; log full details server-side only.

====================================================
4) CODEBASE STRUCTURE / MAINTAINABILITY ISSUES
====================================================

4.1 Duplicate JS + TS source files in same modules
- Examples:
  - Admin_Client(Admin Frontend)/src/App.tsx and App.js
  - Admin_Client(Admin Frontend)/src/main.tsx and main.js
  - Admin_Client(Admin Frontend)/src/components/common.tsx and common.js
  - Admin_Client(Admin Frontend)/src/types/index.ts and index.js
- Impact: Drift risk, confusion, accidental imports from wrong version.
- Improvement: Keep single source of truth (prefer TS), remove generated/legacy duplicates.

4.2 Backup and legacy files committed in active codebase
- Examples:
  - Admin_Client(Admin Frontend)/src/pages/Dashboard_backup.tsx
  - User_Client(User Frontend)/src/pages/admin/Dashboard_backup.tsx
  - Backend/server_backup.js
- Impact: Noise and maintenance risk.
- Improvement: Move to archive/docs or remove from active tree.

4.3 Mixed React Router major versions across apps
- User client package: react-router-dom v7.x
- Admin client package: react-router-dom v6.x
- Impact: Inconsistent routing patterns and higher maintenance burden.
- Improvement: Standardize router version and route conventions across frontends.

4.4 Nested Routes pattern in User App is harder to reason about
- File: User_Client(User Frontend)/src/App.tsx
- Issue: Route path="*" renders a nested <Routes> tree inside element.
- Impact: More complex route debugging and 404 handling.
- Improvement: Flatten route definitions and add explicit NotFound route.

6) ASSET / DEPLOYMENT RISKS
====================================================

6.1 Case inconsistency in image paths
- Several pages use /Images/... while others use /images/...
- Public folder observed as images/ (lowercase).
- Impact: Works on case-insensitive systems, can fail on Linux/case-sensitive hosting.
- Improvement: Normalize all asset paths and folder naming to one case.

6.2 Large bundle warnings in both frontends
- User build output includes very large chunk (>2MB uncompressed main chunk).
- Admin build also warns about chunks >500KB.
- Impact: Slower load time / poorer performance.
- Improvement: Route-level code splitting, dynamic imports, and manual chunk strategy.
